# JavaMaster研修について

## 🧭 研修用レポジトリについて

このレポジトリは JavaMaster 研修用です。

日々の課題・解説で作成するファイルは、 `lecture1`, `lecture2`
のようなパッケージの中に追加していきます（講師の指示に従ってください）。

解説用パッケージの命名規則は課題番号に対応します。

    例：
    - kadai3 → lecture3
    - kadai10 → lecture10

------------------------------------------------------------------------

## 🧑‍💻 Gitの運用ルール（必須）

### ■ 基本ルール

-   1 step 完了ごとに commit を行う
-   最後の step 完了後に push を行う
-   その日の作業は「17:00以降の push」で区切る

------------------------------------------------------------------------

### ■ commitルール

commit メッセージは必ず以下の形式にする

    jm10-step02

形式：

    課題番号-step番号

------------------------------------------------------------------------

### ■ 基本フロー（毎回共通）

``` bash
git status
git add .
git status
git commit -m "jm10-step01"
```

※ `git status` で変更内容を必ず確認してから commit すること

------------------------------------------------------------------------

### ■ pushルール（1日1回）

その日の最後の step が完了したら push を行う

``` bash
git push origin main
```

------------------------------------------------------------------------

## 📖 Git状態確認コマンド

### ■ 履歴を確認する場合

``` bash
git log --oneline
```

------------------------------------------------------------------------

## ⚠️ ブランチ名が master の場合（初回のみ）

git-bash の表示が `(master)`
になっている場合は、以下を実行してください。

``` bash
git branch -m master main
git config --global init.defaultBranch main
```

実行後は `(main)` と表示されるようになります。

------------------------------------------------------------------------

## 💡 補足

-   commit は「作業の区切りを記録する操作」
-   push は「ローカルの履歴を共有する操作」
-   git status は「今の状態を確認する操作」
